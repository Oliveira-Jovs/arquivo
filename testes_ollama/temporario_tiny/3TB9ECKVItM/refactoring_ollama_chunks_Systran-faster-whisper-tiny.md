Sejam muito bem-vindos a mais uma aula sobre linguagem de programação específica para abordar conceitos de scheduling. O conceito de hoje será o Onroll, que, como visto em exemplos anteriores, pode ser utilizado para diminuir o número de iterações ao custo do tamanho do corpo do loop.

Nós vamos aumentar o corpo do loop a fim de diminuir o número de gerações. É possível ver que ele tem um efeito de redução no número de iterações quando comparado à abordagem tradicional, transformando múltiplas iterações em uma só dentro do loop.

Outro caso de uso para Onroll é quando se tem mais de um canal e é necessário executar operações nesses canais em sequência, um após o outro, sem criar novos loops. O Onroll tem duas sintaxes: a primeira envolve passar um array para dentro do Onroll, e a segunda utiliza um fator que pode ser aplicado com um bão ou com o Split.

No caso de querer fazer um Onroll de um array que não seja os canais, geralmente é necessário fazer um Split desse array e, em seguida, aplicar o Onroll na variável resultante, que vai estar alinhada com o corpo do loop. Uma outra opção é utilizar a sintaxe do Onroll passando o array e depois o fator, mas isso pode esconder o Split necessário para alguns casos, como quando se trabalha com canais.

Nós vamos utilizar tanto o Rui Order quanto o bão de para mover os canais para a parte mais interna do loop. O bão de é usado para especificar que a variável do canal é limitada entre zero e três, indicando que se trata de um sinal RGB. Ao trabalhar com Rui Order, é importante considerar a extensão da limensão, permitindo passar a variável, o mínimo (zero) e o existente (três), sabendo os limites do prêmico rei.

A utilização do bão de permite passar essa informação para um rol posterior, especialmente quando se sabe a dimensão do loop. No caso de variáveis que não são canais, pode ser necessário utilizar uma abordagem diferente, como o Splitim, para alcançar o mesmo objetivo.

Um exemplo prático é aplicar um efeito sépio em uma imagem. Inicialmente, mostramos o protótipo em Python, que tem uma velocidade de 20 mil segundos na média, podendo chegar a 46 mil segundos no pior caso. O protótipo envolve multiplicar os canais da imagem por diversos fatores e aplicar um margin desses canais.
