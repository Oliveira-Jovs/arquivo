As pessoas que trabalham em uma equipe de desenvolvimento de software com frequência têm que integrar, juntar as contribuições que elas individualmente fazem para a construção do sistema. Eles normalmente fazem isso através de comandos como git merge, git rebase, git cherry-pick, que são analisados com dificuldades quando as pessoas tentam mudar a mesma parte do código de um mesmo método, como por exemplo, quando duas pessoas tentam alterar linhas consecutivas de um mesmo programa, de um mesmo método, de uma mesma classe.

Mas muitas vezes essas mudanças feitas individualmente por essas pessoas, apesar de serem conflitantes, não são detectadas por essas ferramentas, que infelizmente são limitadas e passam batidas e integram as contribuições de forma equivocada, o que eventualmente leva a uma falha do sistema quando eles tiverem sido utilizados por seus usuários, trazendo consequências negativas para muita gente. Muitas vezes consequências sérias.

Para evitar essas limitações dessas ferramentas atuais de integração de código, nós, no sigo, SPG, em colaboração com pesquisadores aqui dos Estados Unidos, pesquisamos e desenvolvemos ferramentas complementares que podem ser acopladas ao git para detectar esses problemas, esses conflitos e assim evitar que eles cheguem aos usuários finais. A gente faz isso usando uma combinação de técnicas de análise estática e dinâmica de código, que assim conseguem detectar os chamados conflitos semânticos, que são justamente causados quando pessoas diferentes de uma mesma equipe alteram partes diferentes do código, mas essas partes são interdependentes e precisam ser ajustadas antes de serem integradas, só que as pessoas individualmente não têm conhecimento disso.
